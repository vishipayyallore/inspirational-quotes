# Educational Content Rules

**Context**: English language learning through inspirational quotes and structured vocabulary building.
**Priority**: HIGH

## Core principles

### Authenticity and Attribution
- **Quote authenticity**: Preserve original wording and meaning of quotes
- **Proper attribution**: Always include author/source when known
- **Context preservation**: Maintain the original context and intent
- **No fabrication**: Do not create fake quotes or artificial examples

### Clarity and Structure
- **Clear definitions**: Vocabulary definitions should be practical and learnable
- **Contextual examples**: Usage examples should come from real quotes in the collection
- **Progressive learning**: Start with basic concepts and build complexity gradually
- **Consistent formatting**: Follow established templates for all content types

### Plain English and Accessibility
- **Beginner-friendly**: Use simple wording first, then add sophistication
- **Clear explanations**: Explain concepts in everyday language before introducing terminology
- **Practical application**: Focus on real-world usage and learning value
- **Daily practice focus**: Design content for 18-minute daily learning sessions

### Quality Content Creation
- **Educational value**: Every quote should inspire or teach a life lesson
- **Vocabulary relevance**: Words should be useful for English learners
- **Grammar connections**: Link grammar concepts to actual quote usage
- **Motivational purpose**: Content should encourage continued learning

## Content Creation Guidelines

### Quote Curation
- Select quotes that are inspiring, educational, or thought-provoking
- Verify authenticity and attribution when possible
- Organize by themes/categories for easy navigation
- Keep formatting consistent with author attribution in italics

### Vocabulary Development
- Choose words that appear in the quote collection or are commonly useful
- Provide clear, practical definitions
- Include real usage examples from quotes
- Explain context and nuance of word usage

### Grammar Learning
- Start with foundational concepts (nouns, pronouns, sentence structure)
- Progress to intermediate and advanced topics
- Use examples from actual quotes in the collection
- Include practical exercises and applications

### Daily Learning Plans
- Design for 18-minute time-boxed sessions
- Balance vocabulary, grammar, and quote reflection
- Include specific, actionable tasks
- Build skills progressively over time

## Voice and Tone

### Personal Learning Approach
- Write as a personal learning journey
- Use reflective language ("I'm learning...", "I noticed...")
- Focus on practical insights and discoveries
- Avoid formal academic or instructor tone

### Motivational and Educational
- Emphasize inspiration and practical value
- Connect concepts to real-life applications
- Encourage continued learning and growth
- Maintain positive, encouraging tone

## Content Organization

### File Structure
- Follow the repository structure defined in 03-repository-structure.md
- Maintain alphabetical organization for vocabulary
- Use thematic organization for quotes
- Keep consistent naming conventions

### Progressive Learning Path
- Foundations: Basic grammar and vocabulary
- Intermediate: More complex grammar and expanded vocabulary
- Advanced: Sophisticated concepts and nuanced usage
- Practical: Real-world application and communication skills

## Quality Standards

### Review Process
- Verify quote attributions are accurate
- Ensure vocabulary definitions are clear and practical
- Check grammar examples are correct and relevant
- Confirm daily learning plans are achievable in 18 minutes

### Continuous Improvement
- Update outdated or unclear content regularly
- Add new quotes and vocabulary as collection grows
- Refine learning plans based on effectiveness
- Maintain consistency across all files and sections

## Markdown Standards
- Use H2 (`##`) for main topics and H3 (`###`) for subtopics
- Use blockquotes (`>`) for definitions or key takeaways when helpful
- Use tables for comparing concepts or organizing information
- Keep line length reasonable for readability
